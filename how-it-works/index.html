How it Works

What is Federated Learning?

Federated learning is a collaborative approach to training machine learning models across multiple institutions. Instead of transferring data to a central location, each institution trains the model locally. Only model updates (not patient data) are shared and combined. This approach allows collaboration while maintaining institutional responsibility over data.

System Architecture

Fed-BioMed is based on an architecture designed for healthcare environments. It separates local data environments from coordination mechanisms while ensuring secure communication between participants.

Autonomous hospital environments where data remains locally stored and protected. Each node hosts its own datasets, executes local model training, and operates under institutional governance and security policies. Only encrypted model updates, never raw patient data, are shared with the federation server.

The researcher defines the training plan, configures the experiment, and coordinates the federated workflow without directly accessing institutional data. The researcher only receives aggregated model results and metadata needed to monitor the training process.

Workflow

A typical collaborative experiment follows these steps and each step is structured and traceable

1

A researcher defines a training plan.

2

Institutions review and approve participation.

3

Local training is performed within each hospital.

4

Model updates are securely aggregated.

5

The improved model is redistributed.

Security in Practice

When a model is trained:

  • The training happens locally inside the hospital’s secure environment.
  • Only mathematical model updates are sent outside — never raw images or patient records.
  • These updates are encrypted during transmission.

This means that even if someone intercepted the communication, they could not read or reconstruct patient data.
